# Matric Nejma 6 - تطبيق ماتريك نجمة 6 للبث المباشر

المصدر الرسمي لتطبيق Matric Nejma 6 (ماتريك نجمة 6) لمشاهدة البث المباشر لكرة القدم، الآن مع نظام مدونة احترافي ولوحة تحكم مُعاد بناؤها.

## المميزات

- 📱 **تصميم متجاوب بالكامل:** يعمل بسلاسة على جميع الأجهزة (هواتف، أجهزة لوحية، حواسيب) مع مظهر موحد.
- 🎨 **نظام ألوان متناسق:** استخدام صارم لـ الأحمر، الأسود، والأبيض عبر التطبيق بالكامل.
- 📝 **نظام مدونة احترافي:**
    - **المقالات:** إدارة كاملة للمقالات مع دعم التصنيفات والوسوم.
    - **التصنيفات والوسوم:** تنظيم المحتوى لسهولة التصفح والبحث.
    - **التعليقات:** نظام تعليقات متكامل مع إمكانية الموافقة عليها من لوحة التحكم.
    - **البحث:** وظيفة بحث قوية للمقالات.
- 🔐 **لوحة تحكم إدارية مُعاد بناؤها:**
    - **لوحة معلومات (Dashboard):** نظرة عامة على الإحصائيات الرئيسية للموقع.
    - **إدارة المستخدمين:** (مستقبلًا: إضافة إدارة المستخدمين).
    - **إدارة المدونة:** إنشاء، تعديل، حذف المقالات، وإدارة التعليقات.
    - **التحليلات:** إحصائيات مفصلة عن المقالات والرسائل.
- 📬 **نموذج اتصال:** مع حفظ الرسائل في قاعدة البيانات وإدارتها من لوحة التحكم.
- 🛡️ **أمان مُحسّن:** حماية ضد XSS، CSRF، وتحقق من صحة المدخلات.
- 🚀 **جاهز للإنتاج:** روابط غير مكسورة، أصول كاملة، وصفحات مكتملة.

## الإعداد والتثبيت

### 1. إعداد قاعدة البيانات (Supabase)

يتطلب المشروع قاعدة بيانات Supabase لتخزين وإدارة البيانات. قم بتشغيل ملف SQL التالي في محرر SQL الخاص بـ Supabase لإنشاء الجدول المطلوب:

```sql
-- Matric Nejma 6 — Supabase one-time setup
create table if not exists public.kv_store (
  key text primary key,
  value jsonb not null,
  updated_at timestamptz not null default now()
);

alter table public.kv_store disable row level security;

create index if not exists kv_store_updated_at_idx on public.kv_store (updated_at desc);
```

### 2. متغيرات البيئة

أنشئ ملف `.env` في الجذر الرئيسي للمشروع (أو قم بتعيينها في بيئة الاستضافة) بالمتغيرات التالية:

```env
SUPABASE_URL="https://your-project.supabase.co" # رابط مشروع Supabase الخاص بك
SUPABASE_SERVICE_ROLE_KEY="your-service-role-key" # مفتاح خدمة Supabase الخاص بك
ADMIN_PASSWORD="your-secure-password" # كلمة مرور قوية للوحة التحكم الإدارية
```

**ملاحظة:** `ADMIN_PASSWORD` هي كلمة المرور للوصول إلى لوحة التحكم. يوصى بشدة بتغييرها إلى كلمة مرور قوية.

### 3. التثبيت والتشغيل المحلي

لتشغيل المشروع محليًا، اتبع الخطوات التالية:

```bash
# استنساخ المستودع
git clone https://github.com/bestwall2/Matric-Nejma6.git
cd Matric-Nejma6

# تثبيت التبعيات (إذا كنت تستخدم npm أو yarn)
npm install # أو yarn install

# تشغيل الخادم
node server.js
```

سيتم تشغيل التطبيق على `http://localhost:5000` (أو المنفذ المحدد في متغيرات البيئة).

## النشر على Vercel

تم تكوين المشروع للعمل بسلاسة مع Vercel. تأكد من إضافة متغيرات البيئة (`SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `ADMIN_PASSWORD`) في إعدادات مشروعك على لوحة تحكم Vercel.

## لوحة التحكم

يمكن الوصول إلى لوحة التحكم الإدارية عبر المسار: `/admin.html`.

**كلمة المرور الافتراضية:** `admin123` (يجب تغييرها فورًا عبر متغير البيئة `ADMIN_PASSWORD`).

## الأمان والتعامل مع الأخطاء

تم تطبيق الإجراءات الأمنية التالية:

- **حماية XSS (Cross-Site Scripting):** يتم تنقية جميع المدخلات التي تعرض على الواجهة الأمامية لمنع هجمات XSS.
- **حماية CSRF (Cross-Site Request Forgery):** تم إضافة فحص أساسي لـ `Origin` و `Referer` للطلبات التي تغير الحالة لمنع هجمات CSRF.
- **التحقق من صحة المدخلات:** يتم التحقق من صحة جميع المدخلات من المستخدم على جانب الخادم لضمان سلامة البيانات.
- **التعامل مع الأخطاء:** تم تحسين التعامل مع الأخطاء على جانب الخادم لتقديم رسائل خطأ واضحة وتجنب كشف معلومات حساسة.

## التوافق مع الأجهزة المحمولة والتصميم

تم تحسين التطبيق بالكامل ليكون متجاوبًا ويعمل على جميع أحجام الشاشات، من الهواتف الذكية إلى الشاشات الكبيرة. يعتمد التصميم على:

- **CSS Grid و Flexbox:** لتخطيط مرن ومتجاوب.
- **Media Queries:** لتكييف التصميم مع أحجام الشاشات المختلفة.
- **وحدات نسبية:** (rem, em, vw, vh) لضمان قابلية التوسع.
- **نظام الألوان:** استخدام صارم لـ الأحمر، الأسود، والأبيض (`#e11d48`, `#000000`, `#ffffff`) لإنشاء هوية بصرية قوية ومتناسقة عبر التطبيق بأكمله.
